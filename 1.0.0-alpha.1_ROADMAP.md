## Calynda 1.0.0-alpha.1 â€” Release Plan

### Versioning rationale

The jump from 0.4.0 to 1.0.0-alpha.1 signals that the language has a stable core (x86_64 / AArch64 / RV64GC backends, CAR archives, manual memory, typed pointers, closures, unions) and is entering a phase where production-adjacent features are being added. The `alpha` qualifier is honest: threading has no compile-time race detection yet, and the full 1.0 feature set is not yet complete.

## Feature 1 â€” Numeric literal underscores

*(Unchanged from the earlier proposal â€” low-risk, do this first.)*

**Scope:** Tokenizer only. One scan loop in tokenizer_literals.c.

**Rules:**
- `_` is legal anywhere inside an integer or float literal except as the first character, last character, or adjacent to `.`
- Stripped before value conversion; no semantic effect
- Hex, decimal, and float all permitted: `0xFF_00`, `1_000_000`, `3.141_592`

**Stages:** Tokenizer only â€” no AST, parser, sema, IR, or backend changes.

**New tests:** ~8 tokenizer assertions (valid placements parse to the correct numeric value; invalid placements â€” leading, trailing, adjacent to `.`, double underscore `__` â€” are rejected with a lex error).

**Milestone:** Complete in 1 day. Land first, cleanest possible diff.

## Feature 2 â€” Type aliases

*(Expanded from earlier proposal with full stage detail.)*

**Syntax:**

```
type Predicate = (int32) -> bool;
type Grid      = float64[][];
export type OptInt = union { Some(int32), None };
```

Aliases are transparent (structural equivalents), not newtype wrappers. Modifiers `export`, `public`, `private` apply. Aliases may not be circular.

**Stages touched:**

| Stage | Change |
|-------|--------|
| Tokenizer | `type` keyword already exists as `TOK_TYPE`? Check; if not, add it. |
| Parser | Parse `TypeAliasDecl` at top-level: `["export"] "type" IDENT "=" Type ";"`. Add `AST_TOP_LEVEL_TYPE_ALIAS` to the top-level dispatch. |
| AST | `AstTypeAliasDecl { char name[]; AstType *aliased_type; bool is_exported; AstSourceSpan span; }` |
| Symbol table | New `SYMBOL_KIND_TYPE_ALIAS`; registered during `st_add_top_level_decl`. Store a pointer to the `AstType` as the alias body. |
| Type resolution | When resolving a named type whose symbol is `SYMBOL_KIND_TYPE_ALIAS`, recurse into the alias body with a visited-set for cycle detection. Same pattern already used for union generic parameters. |
| Type checker | No new logic â€” aliases are already fully resolved before type checking runs. |
| Semantic dump | Render alias declarations in scope output: `type Predicate = (int32) -> bool` |
| HIR / MIR / Backends | No changes. |

**EBNF change (`calynda_v2.ebnf`):**

```ebnf
TypeAliasDecl ::= { Modifier } "type" Identifier "=" Type ";" ;
TopLevelDecl  ::= ... | TypeAliasDecl ;
```

**New tests:** ~5 parser tests, ~5 type-resolution tests (transparent assignment compatibility, cycle rejection), ~3 type-checker tests (alias as parameter type, alias as return type, generic alias), 1 semantic dump test, 1 E2E native test.

## Feature 3 â€” Multithreading

This is the primary differentiator for 1.0.0-alpha.1.

### 3.1 Design goals and constraints for alpha.1

- Minimal but complete: spawn a thread, wait for it, protect shared state
- Java-style syntax (consistent with existing philosophy)
- No compile-time data race detection â€” that is a 1.0.0-beta goal
- All backends (x86_64, AArch64, RV64GC) gain threading automatically via the existing runtime-helper dispatch path â€” no per-backend code beyond the runtime
- Bytecode portable-v1 records spawn as a known runtime helper call; execution requires a threading-capable runtime

### 3.2 New built-in types

Two new opaque built-in types are added alongside `string`, `bool`, and `ptr<T>`:

| Type | AST kind | Description |
|------|----------|-------------|
| `Thread` | `AST_TYPE_THREAD` | Handle to a running thread |
| `Mutex` | `AST_TYPE_MUTEX` | Mutual exclusion lock |

Both are word-sized opaque handles at the machine level (stored as tagged heap pointers, same as closures and strings). Neither takes generic parameters in alpha.1.

### 3.3 Language surface

**Spawn:**

```
Thread t = spawn () -> {
    // runs concurrently
};
```

`spawn expr` is a new expression. The right-hand side must be a zero-argument callable (type checker enforces this). Return type of `spawn` is always `Thread`. The spawned callable must have `void` return type â€” returning a value from a spawned thread is deferred to 1.0.0-beta (where a `Future<T>` type will be introduced).

**Join:**

```
t.join();
```

`.join()` is a member call on a `Thread` value. It blocks until the thread finishes and returns `void`. Joining an already-joined thread is a runtime abort (same pattern as double-free in `manual checked`).

**Mutex creation:**

```
Mutex m = Mutex.new();
```

`Mutex.new()` uses the existing union-constructor member-access syntax â€” no new grammar needed. The type checker recognizes `Mutex.new()` as the constructor for the `Mutex` built-in type.

**Lock and unlock:**

```
m.lock();
// critical section
m.unlock();
```

`.lock()` and `.unlock()` are member calls on a `Mutex`, both returning `void`. They map directly to the runtime mutex helpers.

**Recommended pattern (library, not new syntax):**

```
// stdlib will expose:
// void with_lock = (Mutex m, () -> void f) -> { m.lock(); f(); m.unlock(); };

with_lock(m, () -> {
    counter = counter + 1;
});
```

`with_lock` as a standard library callable, not a keyword â€” keeping language changes minimal.

### 3.4 Alpha.1 safety notes (documented, not enforced)

- No compile-time data race detection. Two threads can capture the same mutable binding. This is the user's responsibility.
- Thread handles must be joined before the binding goes out of scope. Failing to join is not detected in alpha.1.
- `manual checked {}` bounds registry is **not thread-safe** in alpha.1. Do not use manual-checked scopes from spawned threads.
- The CAR module init function (`__mir_module_init`) runs single-threaded before `start`; global initialization is race-free.

### 3.5 Implementation: compiler stages

| Stage | Change | Notes |
|-------|--------|-------|
| **Tokenizer** | Add `TOK_SPAWN` for `spawn` keyword | 1 token enum entry; 1 keyword string in the keyword table |
| **Parser** | Parse `SpawnExpr ::= "spawn" Expression` as a new expression form at unary precedence | New `AstSpawnExpr { AstExpression *callable; AstSourceSpan span; }` |
| **AST** | Add `AST_EXPR_SPAWN` to the expression kind enum; add `AST_TYPE_THREAD`, `AST_TYPE_MUTEX` to type kinds | The spawn node holds the callable sub-expression |
| **Symbol table** | No new symbol kinds | `Thread` and `Mutex` are resolved by type resolution without symbol table entries (same as `ptr<T>` which is `AST_TYPE_PTR`) |
| **Type resolution** | Handle `AST_TYPE_THREAD` and `AST_TYPE_MUTEX` as zero-arity built-in types â€” no components to recurse into | 2 trivial cases added to the type resolution switch |
| **Type checker** | (1) `spawn expr`: verify expr is a zero-arg void callable â†’ result type `CHECKED_TYPE_THREAD`. (2) Member `.join()` on Thread â†’ void. (3) `Mutex.new()` recognized as Thread constructor â†’ `CHECKED_TYPE_MUTEX`. (4) Member `.lock()`/`.unlock()` on Mutex â†’ void. | Handled in the member-access checker alongside the existing `.tag`/`.payload` special cases |
| **HIR** | Desugar `AstSpawnExpr` into a call to a synthetic `__calynda_rt_thread_spawn` helper; desugar `.join()` into `__calynda_rt_thread_join`; `Mutex.new()` into `__calynda_rt_mutex_new`; `.lock()`/`.unlock()` into `__calynda_rt_mutex_lock`/`__calynda_rt_mutex_unlock` | No new HIR node type needed â€” desugars to existing `HirCall` nodes with known helper symbols. Same pattern as `stackalloc` â†’ `__calynda_stackalloc` |
| **MIR / LIR / Codegen / Machine / AsmEmit** | No changes | Helper calls go through the existing runtime-backed call path |
| **Runtime ABI** | Add 5 new helper signatures to `runtime_abi.h` | `thread_spawn`, `thread_join`, `mutex_new`, `mutex_lock`, `mutex_unlock` |
| **Runtime** | Implement the 5 helpers using pthreads in a new `runtime_threads.c` | pthread_create trampoline calls the Calynda callable dispatch helper; pthread_mutex_t heap-allocated and returned as opaque word |
| **Bytecode** | The 5 runtime calls emit as `BC_CALL` to the well-known `__calynda_rt_*` symbols â€” no new bytecode instructions needed | Bytecode execution requires a threading-capable runtime; documented in bytecode ISA |
| **Build system** | Add `runtime_threads.c` to `RUNTIME_OBJS` in Makefile; link with `-lpthread` for all native-build targets | 2 Makefile lines |

### 3.6 Runtime helper signatures

```c
/* runtime_threads.c â€” new file */

/* Spawn: takes a Calynda callable word (closure or function pointer).
   Returns an opaque Thread handle word. */
calynda_word calynda_rt_thread_spawn(calynda_word callable);

/* Block until thread completes. Aborts on double-join. */
void calynda_rt_thread_join(calynda_word thread_handle);

/* Allocate and initialise a new pthread_mutex_t.
   Returns an opaque Mutex handle word. */
calynda_word calynda_rt_mutex_new(void);

/* Lock / unlock. Both abort on NULL handle. */
void calynda_rt_mutex_lock(calynda_word mutex_handle);
void calynda_rt_mutex_unlock(calynda_word mutex_handle);
```

The thread trampoline inside `calynda_rt_thread_spawn` uses the existing `calynda_rt_callable_dispatch` helper to invoke the passed callable, so all closure-capture semantics work without any additional machinery.

### 3.7 EBNF additions

```ebnf
(* calynda_v2.ebnf additions *)
SpawnExpr    ::= "spawn" UnaryExpression ;
PrimaryExpr  ::= ... | SpawnExpr ;

(* New type terminals *)
Type         ::= ... | "Thread" | "Mutex" ;
```
