# 0.4.0 Remaining Work

This is now a live execution checklist for the 0.4.0 cut. Checked items are already landed in the current tree. Unchecked items still need code, semantics, or release work and should not be treated as done.

## Shared Descriptor Model And Reified Metadata

- [x] Move runtime hetero arrays onto `CalyndaRtTypeDescriptor` metadata with owned cleanup.
- [x] Rework `__calynda_rt_hetero_array_new` to take a descriptor pointer plus an element pack.
- [x] Lower `arr<?>` bytecode construction through shared descriptor constants instead of inline tag arrays.
- [x] Lower native hetero-array construction through static `typedesc(arr|...)` operands.
- [x] Thread `arr<?>` descriptor tags into MIR and LIR so downstream lowering stops rebuilding that metadata ad hoc.
- [x] Cover the new MIR, LIR, bytecode, machine/native, and runtime descriptor-backed constructor paths with regressions.
- [x] Freeze the final runtime descriptor contract for named generic types, wildcard slots, and backend-visible lowering.
- [x] Decide the final amount of generic metadata that survives into MIR, bytecode constants, and native descriptors.
- [x] Remove the remaining union-vs-`arr<?>` metadata split outside constructor syntax by carrying one shared descriptor shape through MIR, LIR, bytecode, and native formatting, with union payload specialization reified in MIR.

## `arr<?>` Beyond Literal Construction

- [x] Land end-to-end literal construction for `arr<?>` across MIR, bytecode, native, and runtime.
- [x] Define indexing semantics for heterogeneous arrays: `arr<?>[i]` reads lower through ordinary index-loads and produce `<external>` values that can be cast, while writes remain disallowed.
- [x] Define mutation rules, especially whether the shape locks after literal creation: `arr<?>` shapes lock after literal construction, indexed reads stay available, and indexed writes remain rejected.
- [x] Define cast and equality behavior: indexed `<external>` values require explicit casts before equality, while `arr<?>` values themselves compare by object identity rather than structural equality.
- [x] Replace the placeholder runtime formatting if 0.4.0 needs more than `<hetero-array:N>`.
- [x] Define how heterogeneous arrays participate in runtime scanning or ownership once the generic metadata model is frozen: the hetero array owns its container, copied descriptor, and element pack, while nested object elements remain independently registry-owned and are reclaimed during runtime cleanup.

## Tagged-Union Parity And Helper Producers

- [x] Keep union construction and `arr<?>` construction on descriptor-backed lowering paths.
- [x] Decide whether tag and payload access become first-class language operations in 0.4.0: yes, union values now expose read-only `.tag` as `int32` and `.payload` as `<external>`, while constructor syntax stays distinct.
- [x] Add real MIR, LIR, bytecode, and machine producers for the union tag/payload helper paths instead of leaving them as unsupported wiring.

## Manual-Memory Hardening

- [x] Decide whether `manual` and `ptr<T>` remain experimental in 0.4.0 or graduate to stable language surface: they now graduate as the stable unsafe manual-memory surface.
- [x] Resolve the `stackalloc` implementation note in `compiler/src/runtime/runtime_manual.c`: `stackalloc(size)` is now documented as scope-local scratch storage reclaimed at manual-scope exit rather than promised machine-stack residency.
- [x] Revisit the fixed checked-manual capacity note in `compiler/src/runtime/runtime_bounds.c`: `manual checked` now uses a growable bounds registry instead of a provisional fixed-cap table.
- [x] Reconcile manual-memory docs and release notes so the experimental contract is explicit.

## Current 0.4.0 Restrictions

- [x] Keep the primitive-only layout-field restriction and document it clearly, or implement broader field-type support before release: 0.4.0 now explicitly rejects non-scalar-primitive layout fields and documents the restriction.
