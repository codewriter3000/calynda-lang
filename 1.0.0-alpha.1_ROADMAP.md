# Calynda 1.0.0-alpha.1 Roadmap Closeout

This file keeps the historical `alpha.1` filename, but the original roadmap is
now closed. The baseline alpha.1 work landed, and the deferred alpha.2
concurrency follow-up is now present in the worktree. The old "deferred to
alpha.2" notes are replaced here with the frozen shipped contract.

## Landed From The Original Alpha.1 Plan

- Numeric literal underscores
- Top-level type aliases
- Baseline `spawn`, `Thread`, and `Mutex`
- Runtime/helper lowering for concurrency through HIR, MIR, native codegen, and
  portable bytecode helper calls
- Cross-target runtime support on x86_64, AArch64, and RV64GC
- MCP/help/doc refresh for the current language surface

## Frozen Alpha.2 Surface

### CLI Contract

- The CLI now exposes version metadata through `calynda --version`.
- The strict race-checker flag name is frozen as `--strict-race-check`.

### Spawn Result Selection

`spawn` still requires a zero-argument callable. Its result now depends on the
callable return type:

- zero-argument `void` callable → `Thread`
- zero-argument non-`void` callable → `Future<T>`

### Supported Member Surface

| Type | Alpha.2 members |
|---|---|
| `Thread` | `.join()`, `.cancel()` |
| `Future<T>` | `.get()`, `.cancel()` |
| `Mutex` | `.new()`, `.lock()`, `.unlock()` |
| `Atomic<T>` | `.new(value)`, `.load()`, `.store(value)`, `.exchange(value)` |

### `thread_local`

`thread_local` now exists, but it is intentionally narrow in alpha.2. It is a
storage feature with cross-thread identity rules, not a general-purpose way to
turn ordinary stack locals into thread-local state.

### `Atomic<T>`

`Atomic<T>` is an alpha.2-only first-class runtime surface for single-word
values. The supported operations are intentionally minimal: construct, load,
store, and exchange.

### Cancellation Semantics

Cancellation uses pthread cancellation semantics.

That means:

- `Thread.cancel()` and `Future<T>.cancel()` force cancellation through the
  pthread model.
- forced cancellation does **not** yet guarantee Calynda-level cleanup
  execution for `manual { ... };` or `manual checked { ... };` scopes.

### `boot()` Contract

The preferred RV64 `boot()` direction landed as preserving the freestanding
promise. Bare-metal `_start` no longer documents Linux-only exit behavior as
part of the contract.

### Backend And Bytecode Note

Alpha.2 concurrency remains helper-based:

- no new bytecode opcodes were introduced for threads, futures, mutexes, or
  atomics
- the surface lowers through existing helper-call paths
- portable execution still depends on a runtime that implements the
  `__calynda_rt_*` helper set

## Post-Alpha.2 Non-Goals

The following are still intentionally out of scope after alpha.2:

- full compile-time ownership or race-proofing beyond the strict race checker
- multi-word or composite atomics
- richer future combinators beyond `get()` / `cancel()`
- broader `thread_local` forms beyond storage with cross-thread identity
- guaranteed Calynda cleanup execution during forced pthread cancellation
